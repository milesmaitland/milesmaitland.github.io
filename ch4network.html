Term Definition
LAN Local Area Network - A network that connects computers and devices in a limited area, such as a home, office, or building.
MAN Metropolitan Area Network - A network that covers a larger geographical area than a LAN but smaller than a WAN.
WAN Wide Area Network - A network that covers a broad area, often connecting multiple LANs and MANs across cities, countries, or even continents.
bus network A network topology where all devices are connected to a single central cable (the bus).
star network A network topology where all devices are connected to a central hub, which manages and controls the network.
CSMA/CD Ethernet Carrier Sense Multiple Access with Collision Detection - A protocol used in Ethernet networks to manage how data is transmitted and handle collisions.
CSMA/CA Wifi Carrier Sense Multiple Access with Collision Avoidance - A protocol used in wireless networks to avoid collisions during data transmission.
repeater A network device that receives and retransmits a signal to extend the reach of a network.
bridge A device used to connect two separate LANs and manage traffic between them.
switch A network device that forwards data packets between devices within a LAN.
router A device that forwards data packets between computer networks, typically connecting a local network to the internet.
gateway A node in a computer network that serves as an entry/exit point to another network.
P2P: peer-to-peer A network model where each node can act as both a client and a server, sharing resources directly with others in the network.
distributed system A network where components are spread across multiple locations, working together as a single system.
cloud computing A method of delivering computing services over the internet, providing storage, processing power, and more.
Tier 1 A high-speed network that forms the backbone of the internet, operated by large companies or organizations.
Tier 2 Regional networks connected to Tier 1 networks, serving as a link between smaller networks and Tier 1 networks.
Tier 3 Local Internet Service Providers (ISPs) and smaller networks that connect to Tier 2 networks to gain access to the internet.
ICANN https://www.icann.org The Internet Corporation for Assigned Names and Numbers - Responsible for coordinating and managing the internet's domain name system and IP addresses.
DNS Domain Name System - Converts domain names to IP addresses, allowing users to access websites by name.
TLD: Top-level domain The highest level in the hierarchical Domain Name System, identifying the most general part of a domain.
Country-code TLDS https://www.worldstandards.ru/other/tlds/ Top-level domains reserved for specific countries or territories, such as .us for the United States or .uk for the United Kingdom.
FTP File Transfer Protocol - A standard network protocol used to transfer files between a client and a server on a computer network.
HTTP Hypertext Transfer Protocol - A protocol used for transmitting web pages over the internet.
SMTP Simple Mail Transfer Protocol - A protocol for sending emails between servers.
URL Uniform Resource Locator - An address used to identify resources on the internet, specifying the location of a web page, file, or document.
client-side activities Tasks performed on the user's device within a client-server model, such as displaying web content or executing scripts.
server-side activities Tasks performed on the server within a client-server model, like processing requests, managing databases, and generating dynamic content.
application layer The layer in the OSI model that provides interfaces for user applications to access network services.
transport layer The layer in the OSI model responsible for end-to-end communication, ensuring data integrity and error correction.
network layer The layer in the OSI model responsible for routing and forwarding data packets to different networks.
link layer The layer in the OSI model responsible for node-to-node data transfer and error detection within a single network.
packet-switching network A network where data is broken into packets for transmission, allowing more efficient use of network resources.
fault tolerant A system's ability to continue operating with minimal disruption in the event of a failure or error.
redundant Extra components or resources in a system used for backup or to ensure reliability in case of failure.
malware Malicious software designed to harm or infiltrate a computer system or network.
virus A type of malware that self-replicates by inserting its code into other programs or documents.
worm A self-replicating type of malware that spreads across networks without needing to attach itself to other programs.
Trojan horse Misleading software that appears legitimate but performs malicious activities once installed.
spyware Software that secretly gathers user information and sends it to third parties without the user's consent.
phishing A type of cyber attack where fraudulent emails or websites trick individuals into revealing sensitive information.
DDoS attack Distributed Denial of Service attack - An attack that floods a system with an overwhelming amount of traffic, rendering it unavailable.
firewall A security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules.
spam filter A software tool that identifies and blocks unwanted or unsolicited emails (spam).
proxy server An intermediary server that acts as a gateway between users and the internet, enhancing security and performance.
antivirus software Software designed to detect, prevent, and remove malicious software (viruses, worms, etc.) from a computer or network.
cryptography The practice and study of secure communication techniques, often involving encryption and decryption.
public key encryption A cryptographic method that uses two keys (public and private) for secure communication and data transmission.
